1. попробовать увеличить скорость обмена до 19200 бод
+2. залить на SD карту поведения, затем давать команды на их выполнение
-2. заливать только команды (keyframe или просто frames). Только потом начинать движение



формат примерно такой
- начинается передача модели передвижения
- если передаётся угол/пауза, записать в хранилище и вернуть полученное(или хеш). 
- если передаётся команда завершения приёма, то сохранить команды или сгенерить их на основе keyframe 
- команда запуска модели

servoFrames mark1 Transfer
010
0290
03180
p100
servoFrames end
servoFrames go mark1




в общем минимизировать обмен данными при работе.

можно сохранить сгенеренные модели просто масивами и просчитывать только переходы между ними

дальше можно поставить модуль памяти(для записи моделей) и I2C платы управления сервами(для снятия нагрузки с контроллера)

копать в сторону павности движений
